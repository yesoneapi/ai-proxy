# YesOneApi：企业 AI 中转站

## 一、产品定位

YesOneApi 是面向企业内部使用的 AI 中转站与统一接入平台。  
通过兼容 OpenAI 风格的统一 API 端点，企业可以将多家大模型供应商接入到同一网关内，实现统一鉴权、统一配额、统一审计与统一安全治理。

对企业而言，YesOneApi 的核心价值不是“再加一个调用层”，而是把分散的模型接入、成本管理和数据安全要求，收敛成一套可落地、可审计、可持续运营的 AI 基础设施。

## 二、为什么企业需要 YesOneApi

随着业务团队广泛使用大模型，企业通常会遇到以下共性问题：

- 多模型接口标准不一致，接入和维护成本高；
- 员工调用缺乏统一预算控制，费用不可见、不可控；
- 调用日志分散，难以满足内部审计与合规要求；
- 敏感数据在调用链路中存在泄露风险；
- 团队扩张后，账号与权限发放流程复杂，治理成本上升。

YesOneApi 针对这些问题提供企业级方案：一套网关统一管理模型调用，配合分层额度、审批流程、审计追踪与 DLP 防护，帮助企业在“效率、成本、安全”之间取得平衡。

## 三、核心能力

### 1) 统一模型网关：一套 API 对接多模型

YesOneApi 兼容 OpenAI API 规范，支持国内外主流模型渠道接入与路由转发，减少多平台适配工作。  
业务系统仅需维护一个网关地址与一套接入方式，即可灵活选择模型策略，并通过负载均衡提升可用性。

<img width="2910" height="1514" alt="image" src="https://github.com/user-attachments/assets/7dc0da17-10ba-4bd7-ad72-c6d9124fbf1c" />


### 2) 三层 Token 额度限流：把 AI 成本管到“人、Key、模型”

在 v1.2.0 中，YesOneApi 增强了三层额度预算控制与预警机制：

- API Key 维度：控制应用或系统调用上限；
- 员工账号维度：约束个人或岗位使用预算；
- 渠道模型维度：按模型成本差异进行精细化管控。

该机制可以有效避免“单点过量调用”“高价模型滥用”和“费用超预算不可追溯”等常见问题。

<img width="2940" height="1514" alt="image" src="https://github.com/user-attachments/assets/60659ad5-791b-49b3-9d34-9132e154c7d2" />


### 3) DLP 数据泄露防护：从拦截到审计的安全闭环

YesOneApi 内置 DLP 防护能力，支持策略规则配置，对敏感内容进行识别与拦截，并形成可追踪的审计日志。  
这让企业在推进 AI 落地时，不仅“能用”，还“可控、可查、可复盘”。

<img width="2910" height="1514" alt="image" src="https://github.com/user-attachments/assets/0127bf18-c700-40c3-9189-50025c3ba185" />


### 4) 企业级管理能力：从发放到运营的全流程支持

围绕企业管理场景，YesOneApi 提供：

- 员工账号批量导入，降低开通成本；
- 员工 Token 额度申请与管理后台审批发放；
- 用量统计与排行榜，帮助识别高价值使用行为；
- 控制台首页关键指标总览（模型预警、安全拦截等）；
- 多用户隔离、分组与权限控制能力。

<img width="2910" height="3676" alt="image" src="https://github.com/user-attachments/assets/493477b1-316c-4dd1-9c5b-d78af292f0ba" />


### 5) 私有化部署与可扩展架构

YesOneApi 支持企业私有化部署，数据留在企业环境内；  
同时支持多机部署、Redis 缓存、MySQL/PostgreSQL 等组合，满足从中小团队到复杂组织的稳定性与扩展性需求。

## 四、版本升级亮点（v1.2.0）

相较于基础网关能力，v1.2.0 重点升级了“成本治理 + 安全治理 + 组织化运营”：

- 升级企业级中转站管理后台；
- 新增三层 Token 额度预算控制与预警；
- 新增 DLP 防护策略与审计日志；
- 新增员工额度申请与审批发放流程；
- 新增全员 Token 使用统计与排行榜；
- 新增批量导入员工账号；
- 升级控制台首页运营指标看板；
- 新增 DeepSeek-V4-Flash 与 DeepSeek-V4-Pro 模型支持。

## 五、与通用“API 中转”方案的差异

市场上常见的中转方案，普遍强调“统一接口、快速接入、多模型聚合”。  
YesOneApi 在此基础上，更聚焦企业内部治理能力：

- 更强调私有部署与企业内网数据闭环；
- 更强调分层预算控制与组织级成本管理；
- 更强调 DLP 与审计追踪，适配合规场景；
- 更强调员工管理、审批流程和运营看板等企业功能。

如果说通用中转方案解决的是“能不能用多个模型”，  
那么 YesOneApi 解决的是“企业如何长期、规模化、安全地用好 AI”。

## 六、典型适用场景

- 企业内部统一 AI 网关建设（替代多点直连）；
- 多部门/多岗位并行使用 AI 的预算治理；
- 对数据安全和审计有要求的业务场景；
- 需要私有部署、稳定运营和持续扩展的组织；
- 希望快速接入并逐步沉淀企业 AI 运营体系的团队。

## 七、结语

YesOneApi 不只是模型转发层，而是企业 AI 使用的治理中台。  
它将“统一接入、成本可控、数据安全、组织化运营”整合到同一平台，帮助企业在 AI 规模化落地过程中走得更稳、更快、更可持续。
